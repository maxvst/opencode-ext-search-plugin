# Глоссарий

Директория проекта, директория открытого проекта, открытая директория - значение переменной openDir - директория, открытая в редакторе OpenCode, передается в плагин через параметр directory.

Директория команды, директория конфига - значение переменной configDir - директория, в которой находится opencode.json в котором инициализируется данный плагин.

Базовая директория - хранится в переменной basePath - директория, которая вычисляется на основе configDir и параметра root из opencode.json, если root не задан, производится fallback на worktree.

Директории внешних зависимостей, директории поиска, внешние директории - хранятся в массиве resolvedDirs, вычисляются исходя из basePath и параметров root и directories из opencode.json

Основной поиск - поиск, инициированный OpenCode, на хук которого настроен плагин.

Вспомогательный поиск, расширенный поиск - поиск, который запускается плагином по внешним директориям. 

Toast-уведомление - всплывающее уведомление в интерфейсе OpenCode, вызывается через `client.showToast`. Плагин использует его для информирования о проблемах конфигурации и ограничениях работы.

FsHost - абстракция над файловой системой (интерфейс в `fs-host.ts`), предоставляет методы `existsSync`, `readFileSync`, `statSync`, `readdirSync`, `globScan`. `globScan` инкапсулирует выбор реализации glob-поиска (Bun.Glob или walkDir fallback). Используется для мокирования в интеграционных тестах. По умолчанию делегирует вызовы к `fs` модуля Node.js.

_testing - экспортируемый namespace в плагине, предоставляющий функции для настройки тестового окружения: `setFsHost`, `resetFsHost`, `setPluginDirOverride`, `resetConfigState`, `setRgPathOverride`, `resetRgCache`, `resetAll`. Подробнее см. [Внутренняя инфраструктура](scenarios/internal-infrastructure.md#namespace-_testing).

spawn - функция (`process.ts`) для запуска внешних процессов с автоматическим выбором runtime: `Bun.spawn` при доступности, fallback на `child_process.execFileSync`. Возвращает `{ stdout, exitCode }`. Подробнее см. [Внутренняя инфраструктура](scenarios/internal-infrastructure.md#spawn--запуск-внешних-процессов).

IGNORE_TOOLS - множество (`constants.ts`) имён инструментов OpenCode, для которых плагин пропускает обработку в хуке `tool.execute.after`. Содержит: `bash`, `read`, `write`, `edit`, `apply_patch`, `task`, `webfetch`, `websearch`, `codesearch`, `skill`, `question`, `todo`, `batch`, `plan`, `lsp`, `deps_read`. Подробнее см. [Внутренняя инфраструктура](scenarios/internal-infrastructure.md#ignore_tools--набор-игнорируемых-инструментов).

excludePatterns - массив строковых паттернов для исключения файлов и директорий из внешнего поиска. По умолчанию: `["node_modules", ".git", "dist"]`. Паттерны без диких карт проверяют сегменты пути, с дикими картами (`*`, `?`, `[`) — применяют glob-matching к имени файла. Подробнее см. [Обработка grep / glob](scenarios/grep-glob-processing.md#алгоритм-исключений).

configDir, директория конфига - директория, в которой находится `opencode.json`, ссылающийся на данный плагин. Определяется обходом от openDir вверх с поддержкой точного и префиксного сопоставления путей. Подробнее см. [Инициализация плагина](scenarios/plugin-initialization.md#сопоставление-путей-при-поиске-configdir).

Широкий searchPath - searchPath, который не указан, либо совпадает с worktree, openDir или любой директорией на прямом пути от openDir до configDir (включая обе граничные директории). Только при широком searchPath выполняется внешний поиск.

Покрытая внешняя директория - внешняя директория, которая находится внутри области основного поиска (searchPath или worktree, если searchPath не указан). Покрытые директории исключаются из вспомогательного поиска и из подсказок.

isPathInExternalDirs — функция (`paths.ts`), проверяющая, находится ли указанный путь внутри одной из внешних директорий (resolvedDirs). Используется для пропуска внешнего поиска, когда searchPath уже указывает на внешнюю директорию (встроенный поиск и так найдёт файлы там). Возвращает `true`, если путь точно совпадает с одной из resolvedDirs или является её подкаталогом.

isNarrowSearchPath — функция (`output-meta.ts`), проверяющая, является ли searchPath «узким» (т.е. не лежащим на прямом пути от openDir к configDir и не совпадающим с worktree/openDir). При узком пути внешний поиск не запускается, поскольку поиск по узкому пути не покрывает внешние директории.

filterCoveredDirs — функция (`paths.ts`), фильтрующая массив resolvedDirs, исключая директории, совпадающие с searchPath или являющиеся его подкаталогами.

isOnDirectPath - функция (`output-meta.ts`), проверяющая, лежит ли указанная директория на прямом пути между двумя заданными директориями (включая обе граничные).

findParentDir - функция (`search.ts`), определяющая, к какой внешней директории принадлежит файл по его пути. Выбирает наиболее специфичное (длинное) совпадение среди массива внешних директорий.

computeHintDirs - функция (`search.ts`), вычисляющая перечень внешних директорий для подсказки (hint). Исключает директории без результатов и те, все результаты которых уместились в бюджет. Сравнивает общее количество результатов по каждой директории с количеством результатов, попавших в бюджет-ограниченный вывод.

hintDirs - массив строк (путей к директориям), возвращаемый функциями `searchExternalGrep` и `searchExternalGlob`. Содержит директории, в которых найдены результаты, но не все из них уместились в бюджет. Используется для формирования подсказки через `buildHint`.

Авто-permit, автоматическое разрешение (auto-permit) — механизм автоматического одобрения запросов OpenCode на доступ к внешним директориям (`external_directory`), пути которых находятся внутри настроенных внешних директорий (resolvedDirs) или внутри configDir. Поскольку пользователь явно указал внешние директории в конфигурации плагина, это подразумевает согласие на доступ к файлам в этих директориях. Одобрение путей внутри configDir необходимо, так как при включённом `strict_path_restrictions` пути поиска перенаправляются в configDir, а в режиме без git configDir может находиться за пределами Instance.directory, что вызывает запрос `external_directory`. Подробнее см. [Авто-permit](scenarios/auto-permit.md).

Обработчик авто-permit (auto-permit handler) — функция, создаваемая `createAutoPermitHandler`, регистрируется как хук `event` плагина. Перехватывает событие `permission.asked` и автоматически одобряет доступ, если запрошенный путь находится внутри resolvedDirs или configDir. Подробнее см. [Авто-permit](scenarios/auto-permit.md).

`createAutoPermitHandler` — фабричная функция (`auto-permit.ts`), создающая обработчик событий для авто-permit. Принимает resolvedDirs, client и опциональный configDir, возвращает async-функцию для хука `event`. Подробнее см. [Авто-permit](scenarios/auto-permit.md#создание-обработчика).

`shouldAutoApprove` — функция (`auto-permit.ts`), определяющая, следует ли автоматически одобрить запрос разрешения. Проверяет тип разрешения (`external_directory`) и наличие запрошенных путей внутри resolvedDirs или configDir. Подробнее см. [Авто-permit](scenarios/auto-permit.md#логика-принятия-решения).

`isInsideDir` — вспомогательная функция (`auto-permit.ts`), проверяющая, находится ли целевой путь внутри указанной директории (точное совпадение или является подкаталогом). Используется для проверки принадлежности путей к configDir при авто-permit. Подробнее см. [Авто-permit](scenarios/auto-permit.md#проверка-принадлежности-к-configdir).

`extractBaseFromGlob` — функция (`auto-permit.ts`), извлекающая базовый абсолютный путь из glob-паттерна путём удаления суффиксов `/**` и `/*`. Возвращает пустую строку для неабсолютных путей. Подробнее см. [Авто-permit](scenarios/auto-permit.md#извлечение-пути-из-glob-паттерна).

`strict_path_restrictions` — опция конфигурации плагина (boolean, по умолчанию `false`). При значении `true` плагин перехватывает вызовы инструментов `glob` и `grep` через хук `tool.execute.before` и перенаправляет пути поиска, находящиеся за пределами configDir и внешних директорий, обратно в configDir. Подробнее см. [Ограничение путей поиска](scenarios/strict-path-restrictions.md).

`isAllowedPath` — функция (`strict-paths.ts`), проверяющая, находится ли указанный путь внутри configDir или одной из внешних директорий (resolvedDirs). Использует нормализацию через `path.resolve` и префиксное сравнение с учётом разделителя пути. Подробнее см. [Ограничение путей поиска](scenarios/strict-path-restrictions.md#логика-проверки-путей).

`createStrictPathBeforeHook` — фабричная функция (`strict-paths.ts`), создающая обработчик для хука `tool.execute.before`. Принимает configDir, resolvedDirs и openDir. Обработчик перехватывает вызовы `glob`/`grep` и перенаправляет `output.args.path` на configDir, если путь не проходит проверку `isAllowedPath`. Подробнее см. [Ограничение путей поиска](scenarios/strict-path-restrictions.md#создание-обработчика).

ExternalDir — тип, описывающий внешнюю директорию с метаданными. Содержит поле `path` (абсолютный путь), `source` (источник: `"config"` — из параметра `directories` в opencode.json, `"compile_commands"` — извлечена из `compile_commands.json`, или `"user"` — из `.ext-search.json`) и опциональное поле `disabled` (если `true`, директория исключается из поиска). Подробнее см. [Поддержка compile_commands.json](scenarios/compile-commands.md), [Пользовательские директории из .ext-search.json](scenarios/user-dirs.md).

`compile_commands_dir` — опция конфигурации плагина (строка, опциональная). Относительный путь к директории с файлом `compile_commands.json` относительно configDir. Если указан, плагин извлекает уникальные директории исходных файлов из базы компиляции и использует их как внешние директории для поиска. Позволяет автоматически определять внешние зависимости без явного перечисления в `directories`. Подробнее см. [Поддержка compile_commands.json](scenarios/compile-commands.md).

source — поле типа ExternalDir, определяющее происхождение внешней директории: `"config"` — задана вручную через параметр `directories`, `"compile_commands"` — извлечена автоматически из `compile_commands.json`, `"user"` — указана в файле `.ext-search.json`.

disabled — опциональное поле типа ExternalDir (boolean). Устанавливается в `true`, если config-директория является поддиректорией cc-директории или если директория (config или cc) находится внутри одной из user-директорий (чтобы избежать дублирования результатов поиска). Disabled-директории исключаются из `activeDirPaths` и не участвуют во вспомогательном поиске, авто-permit и других операциях.

`parseCompileCommands` — функция (`compile-commands.ts`), парсящая `compile_commands.json` и возвращающая массив ExternalDir с `source: "compile_commands"`. Принимает относительный путь к директории, configDir и массив уже разрешённых config-директорий. Пропускает директории внутри configDir и config-внешних директорий, выполняет дедупликацию с исключением вложенных. Возвращает `{ dirs, error? }`. Подробнее см. [Поддержка compile_commands.json](scenarios/compile-commands.md#парсинг-compile_commandsjson).

`markDisabledConfigDirs` — функция (`compile-commands.ts`), помечающая config-директории как `disabled: true`, если они находятся внутри одной из cc-директорий. Это предотвращает дублирование результатов: если cc-директория покрывает более широкую область, специфичная config-директория внутри неё исключается. Подробнее см. [Поддержка compile_commands.json](scenarios/compile-commands.md#пометка-config-директорий-как-disabled).

`.ext-search.json` — конфигурационный файл, расположенный в configDir (рядом с `opencode.json`). Содержит JSON-объект с полем `user_dirs` — массивом строковых путей к директориям, которые следует использовать как внешние. Позволяет добавлять внешние директории без изменения `opencode.json`. Пути поддерживают абсолютные, относительные (от basePath) и домашние (`~/`) форматы. Подробнее см. [Пользовательские директории из .ext-search.json](scenarios/user-dirs.md).

`parseUserDirs` — функция (`user-dirs.ts`), парсящая `.ext-search.json` и возвращающая массив ExternalDir с `source: "user"`. Принимает configDir, basePath, массив уже разрешённых директорий и абсолютный путь configDir. Выполняет валидацию: проверку типа (string), существование директории, отсутствие вложенности в configDir, отсутствие конфликтов с существующими директориями. Выполняет дедупликацию через `addDirNoNested`. Возвращает `{ dirs, errors, parseError? }`. Подробнее см. [Пользовательские директории из .ext-search.json](scenarios/user-dirs.md#парсинг-ext-searchjson).

`markDisabledByUserDirs` — функция (`user-dirs.ts`), помечающая config- и cc-директории как `disabled: true`, если они находятся внутри одной из user-директорий. Это предотвращает дублирование результатов: user-директория покрывает более широкую область, и вложенные директории других источников исключаются. Подробнее см. [Пользовательские директории из .ext-search.json](scenarios/user-dirs.md#пометка-директорий-как-disabled-при-наличии-user-директорий).
