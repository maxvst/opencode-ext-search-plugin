# Сценарии

## Основной сценарий
1. Плагин регистрирует хук tool.execute.after и обрабатывает grep и glob
2. Если данные tools производят поиск в директории проекта (точное совпадение), плагин производит поиск с теми же условиями во внешних директориях.
3. Результат поиска подмешивается к ответу изначально вызванного tool.
4. К ответу добавляется перечень внешних директорий.

## Обработка ошибок

### Отображение toast ошибок
Плагин должен показать toast через opencode api c возникшей проблемой в следующих случаях:
- не найдена утилита rg
- ошибка парсинга opencode.json
- не найден opencode.json, инициализирующий данный модуль
- пустой список внешних директорий
