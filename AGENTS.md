# Краткое описание

Данный проект представляет собой плагин для ИИ-код-ассистента OpenCode.
Этот плагин облегчает поиск файлов в большой монорепе и позволяет при открытии небольшого подпроекта получать доступ к файлам из других подпроектов.

# Ограничения

**ВАЖНО** данный плагин должен уметь ориентироваться в монорепе в которой не установлен git и соответственно нет директории .git или каких-либо других git-артефактов.

В e2e тестах **ЗАПРЕЩЕНО** инициализировать git репозиторий или каким либо еще способом добавлять git-артефакты в том числе и директорию .git 

**ЗАПРЕЩЕНО** каким либо образом менять тесты без явного указания, в том числе изменять конфигурацию, удалять или перемещать их. Если выполнение задачи подразумевает изменение тестов, но явного указания не было, необходимо запрость разрешение черз question tool с обоснованием необходимости изменений и с детальным описанием производимых изменений.

**ЗАПРЕЩЕНО** использовать git команду без явного разрешения. Если разрешения нет, а использование необходимо, запроси его через tool question с обоснованием необходимости использования, так же покажи точные аргументы, с которыми будет вызваться git. **НЕОБХОДИМО** запрашивать разрешение на каждый запуск команды git в отдельности.

# Требования к сопровождению проекта
При добавлении нового функционала или изменении текущего, если об этом не сказано явно в задании, **предложи через 2 отдельных вызова question tool**
1. обновить документацию в директории `docs` с указанием перечня вносимых изменений.
2. написать или доработать тесты, покрывающие новую функциональность.

# Требования к написанию документации
1. Старайся использовать терпины из `docs/glossary.md`
2. Сохраняй структуру описания сценариев, в случае необходимости создавай новые файлы в `docs/scenarios/`, давай при этом ссылки на эти файлы из `docs/scenarios.md` и `README.md`

# Требования к написанию тестов
Если в задании не сказано явно, покрывай функционал тестами в следующем приоритете:
1. Старайся использовать e2e тесты - они самые предпочтительные
2. Используй интеграционные тесты, если написание e2e тестов невозможно или затруднительно (например, тесты, требующие в проекте сотни файлов), либо в e2e тестах уже проверен крайне похожий сценарий.
3. Используй юнит тесты, если написание дополнительного интеграционного теста приведет к появлению практически одинаковых тестов.

# Дополнительная информация

Исходные коды OpenCode в случае необходимости можно посмотреть в `../opencode`.
**ЗАПРЕЩЕНО** в коде каким либо образом ссылаться на код из директории `../opencode`, например подключать из нее библиотеки. Эта директория дана исключительно для ознакомления.
